
# BUG-001: Potential SQL Injection Vulnerability in Login Field

**Severity:** Critical | **Priority:** High
**Reported By:** Shaza Faizer | **Date:** Feb 18, 2026

### Description
The username field accepts SQL injection syntax without proper sanitization. 

### Steps to Reproduce
1. Navigate to the login page.
2. Enter `' OR '1'='1` in the username field.
3. Enter any text in the password field.
4. Click "Login".

### Expected Result
The system should sanitize the input and return a generic "Invalid Credentials" error.

### Actual Result
The system processes the query, posing a significant security risk to the database.
